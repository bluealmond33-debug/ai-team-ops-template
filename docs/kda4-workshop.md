# KDA4 공유용 안내

## 한 줄 소개

AI 에이전트를 챗봇처럼 쓰지 않고, 작은 팀처럼 운영하기 위한 GitHub 템플릿입니다.

## 이런 분께 유용합니다

- 팀 프로젝트에서 AI를 여러 역할로 나눠 쓰고 싶은 분
- GitHub Issue/PR과 AI 작업을 연결하고 싶은 분
- 공부 기록, 결정, 회고를 Obsidian식으로 남기고 싶은 분
- 매번 프롬프트를 새로 짜는 대신 운영 템플릿을 fork하고 싶은 분

## 15분 실습

1. repo fork
2. `QUICKSTART.md` 따라 demo mode 실행
3. `examples/demo-agency/team.yaml`에서 역할 바꾸기
4. `templates/handoff.md`로 작업 요청 1개 작성
5. GitHub Issue로 옮기기
6. 결과를 `examples/sample-obsidian-vault/04 Worklogs/`에 기록
