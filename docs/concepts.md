# Concepts

## Super sample

맛보기 샘플이 아니라, 다른 사람이 fork해서 바로 자기 상황에 맞게 바꿀 수 있는 전체 레시피입니다.

## Handoff

작업을 에이전트에게 넘길 때는 다음이 필요합니다.

- 배경
- 요청
- 제약
- 기대 결과
- 검증 기준
- 돌아올 위치

## Context discipline

긴 작업은 채팅 히스토리에 의존하지 않습니다.

- 시작할 때 상태 파일 작성
- 중간 결과와 근거를 파일에 저장
- 최종 보고는 짧게
- 다음 사람이 이어서 할 수 있게 resume path 기록

## Public-safe template

실제 운영 시스템에서 배운 구조는 공유하되, 실제 개인/팀 데이터는 공유하지 않는 방식입니다.
