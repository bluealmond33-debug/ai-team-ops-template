# 필요한 이미지 목록

README와 KDA4 공유글을 더 쉽게 만들기 위해 필요한 이미지입니다.

## 우선순위 높음

### 1. GitHub Fork 버튼 위치
- 목적: 비개발자가 처음에 어디를 눌러야 하는지 바로 알게 하기
- 형태: GitHub repo 상단 우측 캡처
- 넣을 위치: README의 “이 저장소를 Fork하기” 아래

### 2. AI 팀 운영 흐름 다이어그램
- 목적: 전체 개념을 한 장으로 이해시키기
- 내용: 아이디어 → 작업 요청 → AI 역할 지정 → 작업 진행 → 기록 → 사람 검토
- 형태: 깔끔한 가로형 다이어그램
- 넣을 위치: README의 “전체 흐름” 섹션

### 3. handoff 문서 예시 화면
- 목적: AI에게 일을 어떻게 맡기고 넘겨받는지 보여주기
- 형태: `templates/handoff.md`를 작성한 예시 캡처
- 넣을 위치: README의 “예시 파일부터 바꾸기” 아래

## 우선순위 중간

### 4. GitHub Issue 예시 화면
- 목적: 작업 요청을 어떻게 정리하는지 보여주기
- 형태: sample issue 화면 캡처
- 넣을 위치: QUICKSTART 또는 KDA4 워크숍 문서

### 5. Obsidian식 기록 폴더 예시
- 목적: 결정, 회고, 학습 내용을 어떻게 쌓는지 보여주기
- 형태: 폴더 구조 캡처
- 넣을 위치: README 또는 docs/kda4-workshop.md

## 이미지 제작 기준

- 글자는 크게
- 한 화면에 하나의 메시지만
- 개발자 도구 느낌보다 “작업 흐름 안내서” 느낌
- 민감 정보, 실제 채팅방 이름, 실제 프로젝트 내부 정보는 넣지 않기
- 캡처가 어렵다면 Figma/Keynote로 가짜 화면을 만들어도 됨
